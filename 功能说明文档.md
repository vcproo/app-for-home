# 家庭记账 App 功能说明文档-

## 1. 项目概述 

本项目是一个家庭共享记账 App 的静态交互原型，使用 `HTML + CSS + JavaScript` 实现。当前版本为前端本地原型，数据都维护在 `app.js` 的内存变量中，刷新页面后会恢复为初始示例数据。

主要页面包括：

- 登录 / 注册页
- 创建家庭 / 加入家庭页
- 账单首页
- 记账页
- 分类管理页
- 资产管理页
- 统计页
- 我的 / 家庭成员页

页面切换通过 `data-page` 和 `data-go` 实现，属于单页应用式交互，不依赖后端路由。

## 2. 登录、注册与家庭加入

### 2.1 登录 / 注册

登录 / 注册页提供“登录”和“注册”两个模式。

填写内容：

- 手机号
- 密码

校验逻辑：

- 手机号必须符合 `1` 开头的 11 位手机号格式。
- 密码长度不能少于 6 位。

登录成功后直接进入账单首页。

注册成功后不会直接进入账单首页，而是进入“加入或创建家庭”页面。

### 2.2 创建家庭 / 加入家庭

注册完成后，用户需要先设置自己的昵称。该昵称会用于后续家庭成员展示和记账流水显示。

填写内容：

- 我的昵称
- 邀请码，加入家庭时需要填写

用户有两种方式进入家庭账本：

- 创建家庭：填写昵称后点击“创建”，系统进入账单首页。
- 加入家庭：填写昵称和邀请码后点击“加入家庭”，系统进入账单首页。

当前原型只做前端流程演示：

- 创建家庭不会真实生成后端家庭记录。
- 邀请码不会真实校验，只校验是否填写。
- 昵称会更新当前用户在成员列表和后续记账流水中的显示名称。

## 3. 账单首页

账单首页用于展示家庭账本概览、资产入口、快捷记账、功能模块和最近记录。

### 3.1 顶部入口

页面右上角为“我的”入口，点击后进入家庭成员页。

### 3.2 账本 / 资产滑动卡片

首页顶部有一个横向滑动选项卡，包含两个卡片：

- 共享结余
- 总资产

点击选项卡或横向滑动卡片都可以切换展示。

共享结余卡片展示：

- 当前共享结余
- 总收入
- 总支出

总资产卡片展示：

- 当前未隐藏资产的总额
- “管理资产”按钮

点击“管理资产”进入资产管理页。

### 3.3 快捷记账

点击“记一笔”按钮进入记账页。

底部中间的加号按钮也可以进入记账页。

### 3.4 首页功能模块

当前首页模块包括：

- 分类：进入分类管理页
- 成员：进入我的 / 家庭成员页
- 统计：进入统计页

### 3.5 最近记录

最近记录展示当前内存中的流水数据。

新增记账或资产转账后，会向最近记录顶部插入一条新记录。

## 4. 记账页

记账页支持记录支出和收入。

### 4.1 支出 / 收入切换

页面顶部有“支出 / 收入”切换。

- 选择支出时，保存后的金额为负数。
- 选择收入时，保存后的金额为正数。

### 4.2 金额

用户填写本次记账金额。

提交时校验：

- 金额不能为空。
- 金额必须大于 0。

### 4.3 分类选择

分类来源于分类管理数据。

用户可以选择餐饮、交通、日用等分类。分类管理页新增、修改、删除分类后，记账页分类选项会同步刷新，分类数据为家庭成员共享。

### 4.4 选择账户

记账页有“选择账户”区域，用于选择本次支出或收入关联的资产账户。

账户来源于资产管理页的未隐藏资产。

展示方式：

- 横向滑动卡片
- 当前选中账户有绿色高亮和勾选标记
- 资产过多时不会拉长页面，可以左右滑动选择

隐藏资产不会出现在记账页账户选择中。

保存流水后，最近记录会显示所选账户。

### 4.5 日期和备注

记账页还包含：

- 日期
- 备注

保存时会使用金额、分类、当前用户昵称、账户和备注生成一条流水。记账时要将当前登录的用户信息记录上。

付款人默认使用当前用户昵称。

## 5. 分类管理页

分类管理页用于维护记账分类。分类数据为家庭成员共享

### 5.1 分类列表

页面展示所有常用分类，每条分类包含：

- 分类图标字
- 分类名称
- 本月金额
- 占比
- 修改按钮
- 删除按钮

### 5.2 添加分类

点击“添加”按钮后打开分类编辑表单。

填写内容：

- 分类名称
- 图标字
- 颜色

保存后：

- 新分类加入分类列表
- 记账页分类选项同步更新
- 统计页分类图例同步更新

### 5.3 修改分类

点击分类右侧“修改”按钮后，编辑表单会回填当前分类信息。

保存后：

- 分类名称、图标字、颜色更新
- 如果当前记账页选中的分类被改名，会同步更新选中状态
- 相关页面重新渲染

### 5.4 删除分类

点击“删除”后，该分类从当前分类数据中移除。

删除后：

- 分类列表刷新
- 记账页分类选项刷新
- 统计页图例刷新
- 如果删除的是当前选中分类，则自动选择剩余的第一个分类

## 6. 资产管理页

资产管理页用于管理家庭资产账户。

### 6.1 总资产

页面顶部展示总资产金额。

计算逻辑：

- 只统计未隐藏资产
- 隐藏资产不计入总资产
- 首页总资产卡片与资产管理页总资产保持同步

### 6.2 资产列表

资产列表展示未隐藏资产。

每条资产包含：

- 资产图标字
- 资产名称
- 所属人
- 突出的资产金额
- 余额管理图标按钮
- 转账图标按钮
- 隐藏资产小眼睛按钮

示例资产：

- 中国邮政储蓄卡，100 元，所属人：我
- 建设银行卡，3860 元，所属人：丈夫
- 招商银行，1000 元，所属人：妻子

### 6.3 添加资产

点击“添加”按钮打开资产编辑表单。

填写内容：

- 资产名称
- 资产金额

当前版本添加资产时不需要填写所属人，新增资产默认所属人为“我”。

保存后：

- 新资产加入资产列表
- 首页总资产同步更新
- 记账页账户选择同步更新

### 6.4 余额管理

点击资产行的余额管理图标按钮，打开资产编辑表单。

可以修改：

- 资产名称
- 资产金额

保存后：

- 资产列表更新
- 首页总资产更新
- 如果该账户在记账页被选中，账户名称也会同步更新

### 6.5 资产转账

点击资产行的转账图标按钮，打开资产转账表单。

转账字段：

- 转出账户：自动使用当前点击的资产
- 转入账户：从其他未隐藏资产中选择
- 转账金额

校验逻辑：

- 必须存在可转入账户
- 转账金额必须大于 0
- 转账金额不能大于转出账户余额
- 转出和转入账户不能相同

提交后：

- 转出账户余额减少
- 转入账户余额增加
- 总资产总额保持不变
- 最近记录新增一条“资产转账”流水
- 流水中显示 `转出账户 → 转入账户`

### 6.6 隐藏资产

点击资产行的小眼睛隐藏按钮后：

- 该资产从资产列表移到隐藏资产列表
- 该资产不再计入总资产
- 该资产不再出现在记账页账户选择中

隐藏资产列表展示：

- 资产名称
- 所属人
- 资产金额
- 恢复显示小眼睛按钮

点击恢复显示后：

- 资产回到资产列表
- 重新计入总资产
- 重新出现在记账页账户选择中

## 7. 统计页

统计页用于按时间维度查看收入、支出和结余。

### 7.1 月 / 年维度切换

页面顶部有“月 / 年”切换按钮。

月维度：

- 日期控件显示 `YYYY年M月`
- 左右按钮切换上个月 / 下个月
- 展示本月支出、本月收入、本月结余

年维度：

- 日期控件显示 `YYYY年`
- 左右按钮切换上一年 / 下一年
- 展示本年支出、本年收入、本年结余

### 7.2 核心统计指标

统计页展示三张核心指标卡：

- 支出
- 收入
- 结余

结余计算公式：

```text
结余 = 收入 - 支出
```

当前原型使用 `monthlyStats` 中的示例数据计算月度和年度统计。

### 7.3 支出概览

支出概览包含：

- 支出总额
- 环形图
- 分类图例

分类图例数据来自分类数据中的金额字段。

### 7.4 成员贡献

成员贡献展示家庭成员在当前示例数据中的支出占比和金额。

## 8. 我的 / 家庭成员页

“我的”页面当前用于查看家庭成员和家庭账本概览。

展示内容：

- 家庭账本概览
- 当前成员数量
- 本月流水数量
- 家庭成员列表

成员列表展示：

- 成员头像字
- 成员名称
- 成员关系
- 本月金额

注册后在家庭加入页填写的昵称会更新当前用户在成员列表中的显示名称。

当前版本不包含成员权限管理、邀请码展示、退出家庭等功能。

## 9. 数据联动关系

当前原型中的核心数据都在 `app.js` 中维护。

主要数据：

- `transactions`：最近记录 / 流水
- `categories`：分类
- `members`：家庭成员
- `assets`：资产账户
- `monthlyStats`：月度统计示例数据
- `currentNickname`：当前用户昵称

联动关系：

- 注册成功后进入家庭创建 / 加入流程。
- 创建或加入家庭后，昵称同步到当前成员和后续记账流水。
- 添加、修改、删除分类后，同步刷新分类页、记账页分类、统计页图例。
- 添加或修改资产后，同步刷新资产页、首页总资产、记账页账户选择。
- 隐藏资产后，同步刷新资产页、首页总资产、记账页账户选择。
- 资产转账后，同步刷新资产余额、首页最近记录、首页总资产、记账页账户选择。
- 保存记账后，同步刷新首页最近记录。
- 月 / 年统计切换后，同步刷新统计指标和日期文案。

## 10. 当前原型限制

当前项目是静态前端原型，存在以下限制：

- 数据不持久化，刷新页面会恢复初始示例数据。
- 登录 / 注册不调用真实接口。
- 创建家庭、加入家庭和邀请码校验均为前端流程演示。
- 记账流水没有写入真实数据库。
- 统计数据主要来自内置示例数据。
- 资产、分类、成员数据都在浏览器当前运行状态中维护。

后续如果接入后端，需要将这些内存数据替换为接口数据，并补充用户、家庭、账本、资产、流水、分类等数据表设计。
