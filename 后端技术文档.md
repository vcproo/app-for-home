# 家庭记账 App 后端技术文档

## 1. 技术栈总览

### 后端（Cloudflare 免费层）

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 运行时 | Cloudflare Workers | 边缘无服务器计算，免费层每天 100,000 请求 |
| Web 框架 | Hono | 轻量级 Web 框架，专为 Cloudflare Workers / Edge 设计 |
| 数据库 | Cloudflare D1 | 边缘 SQLite 数据库，免费层 5GB 存储，每日 500 万行读取 |
| 认证 | JWT | 使用 `jose` 库（兼容 Workers 运行时），token 有效期 7 天 |
| 密码加密 | MD5 + Salt | 需求文档要求，每个用户独立随机盐值 |
| 后端语言 | TypeScript | 类型安全，与 Hono 和 Workers 生态天然兼容 |
| API 风格 | RESTful JSON | 统一响应格式，标准 HTTP 方法 |

### 前端（HBuilderX + Uni-app）

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 开发工具 | HBuilderX | DCloud 官方 IDE，内置 Uni-app 编译和云打包 |
| 前端框架 | Uni-app (Vue 3) | 基于 Vue.js 的跨平台框架，一套代码多端运行 |
| UI 组件 | uni-ui / 自定义 | 可使用官方组件库或自定义组件 |
| 状态管理 | Pinia（可选） | Vue 3 官方状态管理，适合管理用户登录态、家庭数据等全局状态 |
| 打包目标 | Android APK | 通过 HBuilderX 云打包生成 APK 安装包 |
| 数据持久化 | uni.setStorageSync | 登录 token、用户偏好等存储在本地 |

### 为什么选择这些技术

- **Hono**: 零冷启动、API 类似 Express、TypeScript 原生支持、文件体积极小（~14KB）
- **D1**: 内置在 Cloudflare 生态中，无需外部数据库服务，支持标准 SQL，免费额度完全够用
- **jose**: Workers 运行时不支持 Node.js 的 `crypto` 模块，`jose` 基于 Web Crypto API，完全兼容
- **不用 KV 做主存储**: 数据需要关系查询和聚合统计，KV 是键值对存储不适合复杂查询
- **Uni-app**: 生态成熟，国内开发者友好，HBuilderX 内置云打包可直接生成 APK，无需配置 Android 环境

## 3. 数据库设计

### 3.1 ER 关系图

```
users (1) ──< family_members >── (1) families (1) ──< categories
                                  |                    |
                                  |                    < assets
                                  |                    |
                                  <────────────────────< transactions
```

一个用户可以加入一个家庭（本设计简化为单家庭），一个家庭拥有多个分类、资产和流水记录。

### 3.2 建表 SQL（迁移脚本）

文件路径：`migrations/0001_init.sql`

```sql
-- 用户表
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    phone TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    salt TEXT NOT NULL,
    created_at TEXT DEFAULT (datetime('now', 'localtime'))
);

-- 家庭表
CREATE TABLE IF NOT EXISTS families (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    invite_code TEXT UNIQUE NOT NULL,
    creator_id INTEGER NOT NULL REFERENCES users(id),
    created_at TEXT DEFAULT (datetime('now', 'localtime'))
);

-- 家庭成员表
CREATE TABLE IF NOT EXISTS family_members (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL REFERENCES families(id),
    user_id INTEGER NOT NULL REFERENCES users(id),
    nickname TEXT NOT NULL,
    relation TEXT DEFAULT '家庭成员',
    role TEXT DEFAULT 'member',
    joined_at TEXT DEFAULT (datetime('now', 'localtime')),
    UNIQUE(family_id, user_id)
);

-- 分类表
CREATE TABLE IF NOT EXISTS categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL REFERENCES families(id),
    name TEXT NOT NULL,
    icon TEXT NOT NULL,
    color TEXT NOT NULL,
    sort_order INTEGER DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now', 'localtime'))
);

-- 资产表
CREATE TABLE IF NOT EXISTS assets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL REFERENCES families(id),
    name TEXT NOT NULL,
    balance REAL DEFAULT 0,
    owner_nickname TEXT NOT NULL,
    hidden INTEGER DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now', 'localtime'))
);

-- 流水表
CREATE TABLE IF NOT EXISTS transactions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    family_id INTEGER NOT NULL REFERENCES families(id),
    user_id INTEGER NOT NULL REFERENCES users(id),
    member_nickname TEXT NOT NULL,
    type TEXT NOT NULL CHECK(type IN ('expense', 'income', 'transfer')),
    amount REAL NOT NULL,
    category_id INTEGER REFERENCES categories(id),
    asset_id INTEGER REFERENCES assets(id),
    transfer_to_asset_id INTEGER REFERENCES assets(id),
    note TEXT,
    transaction_date TEXT NOT NULL,
    created_at TEXT DEFAULT (datetime('now', 'localtime'))
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_transactions_family_date
    ON transactions(family_id, transaction_date DESC);
CREATE INDEX IF NOT EXISTS idx_transactions_family_category
    ON transactions(family_id, category_id);
CREATE INDEX IF NOT EXISTS idx_transactions_family_member
    ON transactions(family_id, member_nickname);
CREATE INDEX IF NOT EXISTS idx_categories_family
    ON categories(family_id);
CREATE INDEX IF NOT EXISTS idx_assets_family
    ON assets(family_id);
CREATE INDEX IF NOT EXISTS idx_family_members_family
    ON family_members(family_id);
CREATE INDEX IF NOT EXISTS idx_family_members_user
    ON family_members(user_id);
```

### 3.3 表结构详解

#### users 用户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 用户唯一 ID |
| phone | TEXT | UNIQUE NOT NULL | 手机号，11 位，作为登录凭证 |
| password_hash | TEXT | NOT NULL | 密码哈希值，`MD5(password + salt)` |
| salt | TEXT | NOT NULL | 随机盐值，注册时生成，32 位随机字符串 |
| created_at | TEXT | DEFAULT datetime | 注册时间 |

#### families 家庭表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 家庭唯一 ID |
| name | TEXT | NOT NULL | 家庭名称，创建时填写 |
| invite_code | TEXT | UNIQUE NOT NULL | 6 位邀请码，随机字母数字组合 |
| creator_id | INTEGER | NOT NULL REFERENCES users(id) | 创建者用户 ID |
| created_at | TEXT | DEFAULT datetime | 创建时间 |

#### family_members 家庭成员表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 记录唯一 ID |
| family_id | INTEGER | NOT NULL REFERENCES families(id) | 所属家庭 ID |
| user_id | INTEGER | NOT NULL REFERENCES users(id) | 用户 ID |
| nickname | TEXT | NOT NULL | 昵称，记账时显示 |
| relation | TEXT | DEFAULT '家庭成员' | 关系：本人/配偶/父母/子女/家庭成员 |
| role | TEXT | DEFAULT 'member' | 角色：owner（创建者）/ member（成员） |
| joined_at | TEXT | DEFAULT datetime | 加入时间 |

**唯一约束**: `(family_id, user_id)` —— 一个用户在同一家庭中只能有一条记录。

#### categories 分类表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 分类唯一 ID |
| family_id | INTEGER | NOT NULL REFERENCES families(id) | 所属家庭 ID |
| name | TEXT | NOT NULL | 分类名称，如"餐饮"、"交通" |
| icon | TEXT | NOT NULL | 图标字，如"餐"、"行"（显示在分类图标中） |
| color | TEXT | NOT NULL | 颜色值，如"#22a84d" |
| sort_order | INTEGER | DEFAULT 0 | 排序序号，值越小越靠前 |
| created_at | TEXT | DEFAULT datetime | 创建时间 |

#### assets 资产表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 资产唯一 ID |
| family_id | INTEGER | NOT NULL REFERENCES families(id) | 所属家庭 ID |
| name | TEXT | NOT NULL | 资产名称，如"招商银行卡" |
| balance | REAL | DEFAULT 0 | 当前余额，通过流水联动更新 |
| owner_nickname | TEXT | NOT NULL | 所属人昵称 |
| hidden | INTEGER | DEFAULT 0 | 是否隐藏：0=可见，1=隐藏 |
| created_at | TEXT | DEFAULT datetime | 创建时间 |

#### transactions 流水表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 流水唯一 ID |
| family_id | INTEGER | NOT NULL REFERENCES families(id) | 所属家庭 ID |
| user_id | INTEGER | NOT NULL REFERENCES users(id) | 记账人用户 ID |
| member_nickname | TEXT | NOT NULL | 付款人昵称，展示用 |
| type | TEXT | NOT NULL CHECK | 类型：expense / income / transfer |
| amount | REAL | NOT NULL | 金额：支出为负数，收入为正数 |
| category_id | INTEGER | REFERENCES categories(id) | 分类 ID，转账类型时为 NULL |
| asset_id | INTEGER | REFERENCES assets(id) | 关联资产 ID |
| transfer_to_asset_id | INTEGER | REFERENCES assets(id) | 转账目标资产 ID（仅 transfer 类型） |
| note | TEXT | | 备注信息 |
| transaction_date | TEXT | NOT NULL | 流水日期，格式 YYYY-MM-DD |
| created_at | TEXT | DEFAULT datetime | 创建时间 |

## 4. API 设计

### 4.0 通用约定

**基础路径**: `/api/v1`

**请求头**:
```
Content-Type: application/json
Authorization: Bearer <jwt_token>  （需认证的接口）
```

**统一响应格式**:
```json
{
  "code": 0,
  "message": "success",
  "data": { }
}
```

**分页参数**（适用于列表接口）:
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | number | 1 | 页码，从 1 开始 |
| page_size | number | 20 | 每页条数，最大 100 |

**分页响应格式**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```

### 4.0.1 错误码定义

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| 0 | 200 | 成功 |
| 1001 | 400 | 手机号格式错误（必须 1 开头的 11 位数字） |
| 1002 | 400 | 密码长度不足（至少 6 位） |
| 1003 | 409 | 手机号已注册 |
| 1004 | 401 | 手机号或密码错误 |
| 1005 | 401 | 未登录或 token 已过期 |
| 1006 | 403 | 无权操作（不属于该家庭） |
| 2001 | 400 | 邀请码无效或不存在 |
| 2002 | 409 | 已经在该家庭中 |
| 3001 | 404 | 分类不存在 |
| 3002 | 404 | 资产不存在 |
| 3003 | 400 | 转账金额超过余额 |
| 3004 | 400 | 不能转给自己（同一资产） |
| 4001 | 404 | 流水记录不存在 |
| 9999 | 500 | 服务器内部错误 |

**错误响应示例**:
```json
{
  "code": 1001,
  "message": "手机号格式错误",
  "data": null
}
```

### 4.1 认证接口

#### 4.1.1 注册

**POST** `/api/v1/auth/register`

请求体：
```json
{
  "phone": "13800138000",
  "password": "123456"
}
```

成功响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": 1,
      "phone": "138****8000"
    }
  }
}
```

业务逻辑：
1. 校验手机号格式（`/^1\d{10}$/`）
2. 校验密码长度 >= 6
3. 检查手机号是否已注册
4. 生成随机 salt（32 位）
5. 计算 `password_hash = MD5(password + salt)`
6. 插入 users 表
7. 签发 JWT token 返回（此时 token 中无 familyId，需后续创建/加入家庭）

#### 4.1.2 登录

**POST** `/api/v1/auth/login`

请求体：
```json
{
  "phone": "13800138000",
  "password": "123456"
}
```

成功响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": 1,
      "phone": "138****8000",
      "family_id": 1,
      "nickname": "我",
      "relation": "本人"
    }
  }
}
```

业务逻辑：
1. 根据手机号查询用户
2. 计算 `MD5(password + user.salt)` 并与数据库中的 `password_hash` 比对
3. 查询该用户是否已加入家庭（family_members 表）
4. 签发 JWT token，payload 中包含 `userId`、`familyId`（如有）、`nickname`（如有）

### 4.2 家庭接口

> 以下接口均需认证（Authorization: Bearer token）

#### 4.2.1 创建家庭

**POST** `/api/v1/family/create`

请求体：
```json
{
  "name": "我的家庭",
  "nickname": "爸爸"
}
```

成功响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "family": {
      "id": 1,
      "name": "我的家庭",
      "invite_code": "A3K9X2"
    },
    "token": "eyJhbGciOiJIUzI1NiIs..."
  }
}
```

业务逻辑：
1. 检查用户是否已有家庭（已有则返回错误 2002）
2. 生成 6 位随机邀请码（大写字母+数字，不含易混淆字符 O/0/I/1/L）
3. 插入 families 表，`creator_id` 为当前用户
4. 插入 family_members 表，role 为 `owner`，relation 为 `本人`
5. 返回新的 JWT token（包含 familyId 和 nickname）
6. **创建默认分类**：餐饮、交通、日用、房租、医疗、教育、娱乐、其他（8 个）

#### 4.2.2 加入家庭

**POST** `/api/v1/family/join`

请求体：
```json
{
  "invite_code": "A3K9X2",
  "nickname": "妈妈"
}
```

成功响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "family": {
      "id": 1,
      "name": "我的家庭"
    },
    "token": "eyJhbGciOiJIUzI1NiIs..."
  }
}
```

业务逻辑：
1. 根据 invite_code 查询家庭
2. 检查用户是否已在该家庭中
3. 插入 family_members 表，role 为 `member`
4. 返回新的 JWT token

#### 4.2.3 获取家庭信息

**GET** `/api/v1/family/info`

成功响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "name": "我的家庭",
    "invite_code": "A3K9X2",
    "member_count": 4,
    "created_at": "2026-06-01 10:00:00"
  }
}
```

#### 4.2.4 退出家庭

**POST** `/api/v1/family/leave`

成功响应：
```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

业务逻辑：
1. 删除 family_members 中该用户的记录
2. 该用户的历史流水记录保留（通过 user_id 关联）
3. 返回新的 JWT token（不包含 familyId）
4. 用户退出后仍可查看自己的账单

#### 4.3.2 新增分类

**POST** `/api/v1/categories`

请求体：
```json
{
  "name": "医疗",
  "icon": "医",
  "color": "#ef5f5f"

  }

```

1. 校验 name、icon、color 非空

#### 4.3.3 修改分类

**PUT** `/api/v1/categories/:id`

请求体：
```json
{
  "name": "医疗保健",
  "icon": "医",
  "color": "#ef5f5f"
}
```

业务逻辑：
1. 校验分类属于当前用户家庭
2. 更新 name、icon、color

#### 4.3.4 删除分类

**DELETE** `/api/v1/categories/:id`

业务逻辑：
1. 校验分类属于当前用户家庭
2. 检查是否有流水关联此分类
   - 有关联流水：将流水的 `category_id` 设为 NULL（保留流水记录）
3. 删除分类

## 5. 认证方案

### 5.1 JWT Token 结构

**Header**:
```json
{
  "alg": "HS256",
  "typ": "JWT"
}
```

**Payload**:
```json
{
  "userId": 1,
  "familyId": 1,
  "nickname": "爸爸",
  "iat": 1717334400,
  "exp": 1717939200
}
```

| 字段 | 说明 |
|------|------|
| userId | 用户 ID |
| familyId | 家庭 ID（未加入家庭时为 null） |
| nickname | 用户在家庭中的昵称（未加入家庭时为 null） |
| iat | 签发时间 |
| exp | 过期时间（签发时间 + 7 天） |

### 5.2 认证中间件

```typescript
// src/middleware/auth.ts
import { Context, Next } from 'hono';
import { verifyToken } from '../utils/jwt';

export async function authMiddleware(c: Context, next: Next) {
    const authHeader = c.req.header('Authorization');
    if (!authHeader || !authHeader.startsWith('Bearer ')) {
        return c.json({ code: 1005, message: '未登录或 token 已过期', data: null }, 401);
    }

    const token = authHeader.substring(7);
    const payload = await verifyToken(token, c.env.JWT_SECRET);
    if (!payload) {
        return c.json({ code: 1005, message: '未登录或 token 已过期', data: null }, 401);
    }

    // 将用户信息挂载到上下文
    c.set('userId', payload.userId);
    c.set('familyId', payload.familyId);
    c.set('nickname', payload.nickname);

    await next();
}
```

### 5.3 JWT 工具函数

```typescript
// src/utils/jwt.ts
import { SignJWT, jwtVerify } from 'jose';

interface TokenPayload {
    userId: number;
    familyId: number | null;
    nickname: string | null;
}

export async function signToken(payload: TokenPayload, secret: string): Promise<string> {
    const key = new TextEncoder().encode(secret);
    return new SignJWT(payload as any)
        .setProtectedHeader({ alg: 'HS256' })
        .setIssuedAt()
        .setExpirationTime('7d')
        .sign(key);
}

export async function verifyToken(token: string, secret: string): Promise<TokenPayload | null> {
    try {
        const key = new TextEncoder().encode(secret);
        const { payload } = await jwtVerify(token, key);
        return payload as unknown as TokenPayload;
    } catch {
        return null;
    }
}
```

### 5.4 MD5 工具函数

```typescript
// src/utils/md5.ts
// Workers 运行时没有 Node.js 的 crypto 模块
// 使用 Web Crypto API 实现简易哈希，或使用轻量 md5 库

// 方案一：使用 SubtleCrypto（SHA-256，非 MD5 但更安全）
export async function hashPassword(password: string, salt: string): Promise<string> {
    const data = new TextEncoder().encode(password + salt);
    const hashBuffer = await crypto.subtle.digest('SHA-256', data);
    const hashArray = Array.from(new Uint8Array(hashBuffer));
    return hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
}

// 方案二：如果必须使用 MD5，引入轻量库
// npm install blueimp-md5
// import md5 from 'blueimp-md5';
// export function hashPassword(password: string, salt: string): string {
//     return md5(password + salt);
// }
```

> **安全建议**: 虽然需求文档要求使用 MD5，但 MD5 已不被认为是安全的哈希算法。建议优先使用 SHA-256（方案一）。如果必须兼容 MD5，请使用方案二。

### 8.3 API 请求封装（Uni-app 版）

创建 `api/index.js`，使用 `uni.request` 替代 `fetch`：

```javascript
// api/index.js

const API_BASE = 'https://app-for-home-worker.your-subdomain.workers.dev/api/v1';

// 获取 token
function getToken() {
    return uni.getStorageSync('token') || '';
}

// 存储 token
function setToken(token) {
    uni.setStorageSync('token', token);
}

// 清除 token（退出登录）
function clearToken() {
    uni.removeStorageSync('token');
}

// 统一请求封装
function request(path, options = {}) {
    return new Promise((resolve, reject) => {
        const url = `${API_BASE}${path}`;
        const header = {
            'Content-Type': 'application/json',
            ...options.header,
        };

        const token = getToken();
        if (token) {
            header['Authorization'] = `Bearer ${token}`;
        }

        uni.request({
            url,
            method: options.method || 'GET',
            data: options.data,
            header,
            success: (res) => {
                const result = res.data;

                if (result.code === 1005) {
                    // token 过期，清除登录态并跳转登录页
                    clearToken();
                    uni.reLaunch({ url: '/pages/login/index' });
                    reject(new Error('登录已过期'));
                    return;
                }

                if (result.code !== 0) {
                    uni.showToast({ title: result.message, icon: 'none' });
                    reject(new Error(result.message));
                    return;
                }

                resolve(result.data);
            },
            fail: (err) => {
                uni.showToast({ title: '网络请求失败', icon: 'none' });
                reject(err);
            },
        });
    });
}

// 便捷方法
export const api = {
    get: (path) => request(path),
    post: (path, data) => request(path, { method: 'POST', data }),
    put: (path, data) => request(path, { method: 'PUT', data }),
    delete: (path) => request(path, { method: 'DELETE' }),
};

export { getToken, setToken, clearToken };
```

### 8.5 本地存储方案（替代 localStorage）

Uni-app 中使用 `uni.setStorageSync` / `uni.getStorageSync` 替代浏览器的 `localStorage`：

| 用途 | 存储 key | 说明 |
|------|----------|------|
| 登录 token | `token` | JWT token，登录后存储 |
| 用户信息 | `userInfo` | `{ userId, familyId, nickname }` 缓存 |
| 登录状态 | `isLoggedIn` | boolean，快速判断是否登录 |

```javascript
// 登录成功后
setToken(data.token);
uni.setStorageSync('userInfo', {
    userId: data.user.id,
    familyId: data.user.family_id,
    nickname: data.user.nickname,
});

// 应用启动时检查登录状态（App.vue）
onLaunch(() => {
    const token = uni.getStorageSync('token');
    if (!token) {
        uni.reLaunch({ url: '/pages/login/index' });
    }
});
```

### 8.7 HBuilderX 本地调试

开发阶段可以在 HBuilderX 中使用内置浏览器或真机调试：

1. **内置浏览器调试**: 直接运行到浏览器，API 请求通过 H5 代理转发
2. **真机调试**: 运行到手机或模拟器，请求直接发往 Workers 地址
3. **本地 Workers 开发**: `wrangler dev` 启动本地服务后，将 API_BASE 改为 `http://localhost:8787/api/v1`

## 10. 免费层限制与注意事项

| 资源 | 免费层限制 | 说明 |
|------|-----------|------|
| Workers 请求数 | 100,000 次/天 | 家庭内部使用完全够用 |
| Workers CPU 时间 | 10ms/请求 | 简单查询足够，避免复杂计算 |
| D1 数据库存储 | 5GB | 足够存储数万条流水 |
| D1 读取行数 | 5,000,000 行/天 | 列表接口注意分页 |
| D1 写入行数 | 100,000 行/天 | 注意批量操作 |
| D1 数据库数量 | 10 个 | 只需 1 个 |
| KV 读取 | 100,000 次/天 | 如需 token 黑名单可使用 |
| KV 写入 | 1,000 次/天 | 一般用不到 |

### 性能优化建议

1. **分页**: 流水列表必须分页，建议每页 20 条
2. **索引**: 已在建表 SQL 中添加常用查询索引
3. **避免 N+1 查询**: 获取流水列表时使用 JOIN 查询分类信息，而非循环查询
4. **统计缓存**: 月度统计数据变化不频繁，可考虑使用 KV 缓存（可选）
5. **资产余额**: 通过 transactions 记录联动更新 assets.balance，避免每次聚合计算

## 11. 安全注意事项

1. **HTTPS**: Cloudflare Workers 默认强制 HTTPS
2. **SQL 注入**: D1 使用参数化查询（`?` 占位符），不会产生 SQL 注入
3. **XSS**: 后端返回 JSON 数据，前端需对用户输入做转义显示
4. **密码安全**: 即使使用 MD5，也必须加 salt；建议升级为 SHA-256
5. **JWT Secret**: 必须使用足够复杂的随机字符串，通过 `wrangler secret` 设置
6. **邀请码**: 不含易混淆字符（O/0/I/1/L），长度 6 位，暴力破解概率极低
7. **权限校验**: 所有写操作必须校验当前用户是否属于目标家庭
8. **数据隔离**: 查询时始终带 `family_id` 条件，防止跨家庭访问
